# AGENTS.md

## 项目目标

本项目是一个“封面风格 → 视频视觉组件库”生成器。核心产物不是完整 App 页面，而是可用于视频剪辑、AE/PR、Figma、Illustrator 和空间高斯交互的独立视觉素材，包括弹窗、贴画、特效文字、空间按钮、信息卡等。

## 核心文件

- `outputs/ElementLab-offline.html`：用户直接双击使用的离线组件库，也是首要交付物。
- `app/page.tsx`：在线 React 版本的页面与交互。
- `app/globals.css`：在线版本的视觉样式。
- `public/cover-reference.png`：封面风格参考图。

## 视觉方向

- 参考前五张图片提取组件类型和排版语言。
- 参考封面图片提取配色与情绪：暖木色、粉色、黑白高对比、粗体描边、杂志拼贴、手写贴纸。
- 避免通用后台管理风格；组件应适合叠加在视频画面或三维空间中。
- 默认透明背景，保证组件可以独立使用。

## 功能要求

- 每个组件必须能单独选择和预览。
- 主文字与辅助文字必须可编辑，并实时反映到预览与导出结果。
- 每个组件必须支持单独导出透明 PNG。
- 每个风格库必须有独立的整套 PNG 导出按钮。
- 顶部必须提供“一键打包当前元素库”快捷按钮，并随风格下拉选择切换目标库。
- 必须支持一键打包全部组件。
- 批量包使用 ZIP，每个组件为独立 PNG 文件。
- 页面内部预览与 PNG 渲染源使用原生 SVG 节点（如 `rect`、`circle`、`path`、`text`），不得使用 `foreignObject`，否则部分环境会显示空白。
- 最终 PNG 必须保持透明背景，默认尺寸为 2400 × 1400。

## 修改规范

- 优先维护 `outputs/ElementLab-offline.html`，确保无需服务器和网络即可运行。
- 新增组件时，同时更新：组件数据、页面预览、原生 SVG 导出模板、分类数量。
- 修改导出功能后，至少验证：内部 SVG 根节点存在、包含可见图形与文字节点、无 `foreignObject`、PNG 渲染函数存在、ZIP 文件包含全部 PNG 组件。
- 不要提交 `node_modules/`、`dist/`、`.wrangler/`、`work/` 等生成或缓存目录。
- 用户可交付的离线文件位于 `outputs/`；该文件即使被 `.gitignore` 忽略，也需要使用强制添加方式纳入版本控制。

## 本地验证

1. 双击 `outputs/ElementLab-offline.html`，或用本地静态服务器打开。
2. 检查 4 个风格库及其全部分类是否显示；每库 20 个组件，总计 80 个。
3. 修改主文字、辅助文字和主题色。
4. 单独导出至少一个透明 PNG，并确认不是空白、尺寸为 2400 × 1400。
5. 分别测试每种风格的整套导出，并确认每个 ZIP 包含 20 个 PNG。
6. 一键打包全部组件，确认 ZIP 内包含 80 个 PNG。
